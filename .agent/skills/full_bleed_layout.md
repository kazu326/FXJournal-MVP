# フルブリード表示時の左右余白（スクロールバーや100vwによるレイアウト崩れ）対策

## 概要
スマートフォン向けのUI（スライド画像など）をパソコン等のブラウザで確認した際、`w-screen` （100vw）で要素を画面幅いっぱいに広げたつもりが、**左右の端が背景色（白枠など）で隠れてしまい、結果的に余白が存在するように見える現象**が発生することがある。本ドキュメントはその原因と修正の知見をまとめたものである。

## 現象と原因

画像自体は設定上広がろうとしているが、以下の2つの要因が組み合わさることで「はみ出た左右の端が隠されてしまう」状態に陥る。

1.  **親コンテナの `overflow-hidden` 制約**
    最上位（あるいは親）のコンテナに `overflow-hidden` が設定されていると、親要素の横幅を超えて広がろうとした子要素（画像など）のはみ出た部分が強制的に切り取られてしまう。
    
2.  **`w-screen` (100vw) と スクロールバーの干渉**
    Tailwind CSS などの `w-screen` (100vw) は、**ブラウザのスクロールバーの幅を含んだサイズ**になってしまう。
    一方、一般的な親要素（bodyなど）はスクロールバーの幅を除いた実際の描画領域の幅しか持たないため、`100vw`の要素は親要素よりもわずかに大きくなる。
    この状態で `overflow-hidden` がかかると、親要素からはみ出した左右数ピクセル〜十数ピクセル分が切り取られ、下地の背景色（白など）が見えてしまい、それが「余白（枠）」のように見えてしまう。

## 解決策・修正手順

この問題に直面した場合、以下の手順で原因を取り除きフルブリードを実現する。

### 1. `w-screen` を `w-full` に変更する
画像を画面幅（親の最大幅）にフィットさせる際は、スクロールバー幅を含む `100vw` (`w-screen`) ではなく、親要素に対して100%になる **`w-full`** を使用する。

```diff
- <main className="flex flex-col w-screen relative left-1/2 -translate-x-1/2 bg-white">
+ <main className="flex flex-col w-full relative left-1/2 -translate-x-1/2 bg-white">
```

### 2. 親要素・祖先要素の制約を取り除く
`w-full`にした上で、親や祖先要素が存在する場合は、それらが持つ予期せぬパディングやマージンが影響していないか確認し、削除する。

- 余計な `px-4` 等のパディングを外す。
- 不必要に画像の横幅を制限している `max-w-` 系のクラスを要素から外すか、必要であれば親のコンテナレベルでのみ調整するよう設計を見直す。

### 3. オーバーフロー設定の見直し
全体に `overflow-hidden` をかけるのではなく、必要な箇所（縦スクロールを残したい場合は横方向だけの `overflow-x-hidden` など）に限定することで、不必要なクリッピングを防ぐ。

---
**※備考:**
画像を元の比率のまま崩さず広げるため、`aspect-[16/9]` などの固定比率指定および `object-cover` などの強制トリミングが原因で表示が崩れている（意図しない箇所が切れる）ケースもある。
フルブリードにおいては `h-auto block` 等、画像本来の比率を保ったまま横幅100%になるよう併せて指定すると確実。
