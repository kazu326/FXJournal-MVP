# Mobile UI Guidelines & Project Context

## プロジェクトコンテキスト
常にこの環境と制約を考慮してUI実装・修正を行ってください。

### 1. 確認環境
- デバイス: モバイル（iOS / Android）
- 画面幅: 375〜390px（モバイルファースト）
- 確認方法: 実機、またはブラウザのレスポンシブ（モバイル）エミュレーター

### 2. 技術スタック
- React + TypeScript + Vite
- Tailwind CSS（v4系: `@layer base` などのカスケード仕様に注意、任意値クラス `[...]` 使用あり）
- Supabase / React Router

### 3. UIルール
- **スライド・画像表示**: `w-full` を使用し、画面幅いっぱいのフルブリード表示にする。
- **ヘッダー・ナビゲーション・テキストエリア**: `max-w-[480px] mx-auto` を個別のコンテナに適用し、タブレットやPCでも中央配置＋幅制限が機能するようにする。（※ルート全体を制限しないこと）
- **余白**: 統一して左右 `px-4`（16px）をベースにレイアウトする。
- **フォント階層**:
  - タイトル / 見出し: `16px bold` (`text-[16px] font-bold`)
  - サブ見出し / ボタン等: `15px medium` (`text-[15px] font-medium`)
  - 補足テキスト①: `13px` (`text-[13px]`)
  - 補足テキスト②: `12px` (`text-[12px]`)

### 4. 修正時の注意点・チェックポイント
- **制約の見え方**: モバイル幅（375px等）では `max-w-[480px]` による見た目の変化は発生しません。「モバイルサイズで視覚的に変化が出るか」を常に意識して修正を提示してください。
- **実機確認ファースト**: 変更後は実機による確認を見据え、細かなマージンやレイアウトの意図（なぜその構造にしたか）を明確にします。
- **Tailwindのキャッシュ**: Tailwindの任意値クラス（例: `w-[calc(100%+32px)]` や `h-[...px]`）を追加・変更した場合や、CSSのレイヤリングルール（`@layer base`等）を編集した場合は、開発サーバー (`npm run dev`) の再起動が必要になるケースがあるため、スタイルが反映されない場合は再起動を案内・実行してください。

## 5. フルブリード（全幅）が効かない場合のチェックリスト
UIで画面の両端いっぱいまで広げたい要素（スライド画像など）が枠内に収まってしまう場合は、以下を確認してください。
1. **App.tsx の最外枠に padding が設定されていないか確認** (ルートレベルのコンテナ依存に注意)
2. **親コンポーネントの `overflow-hidden` を確認**
3. **`w-screen` (`100vw`) はスクロールバーの幅を含むため使用禁止** (`w-full` を使用すること)
4. **Tailwindクラスより `style={{ ... }}` 直書きの方が確実な場合がある**（動的な値や最上位のレイアウト調整時）
